## 4️⃣ INSTRUCTIONS DE CONFIGURATION & ACCÈS AUX FONCTIONNALITÉS

Installation:
1. Installer l'application depuis TestFlight (build interne) ou depuis l'App Store si disponible.
2. Ouvrir l'application et attendre le splash screen.

Connexion:
- Utiliser l'un des comptes de test listés ci‑dessus (`test.artist@wmahub.ua`, `test.distributor@wmahub.ua`, `test.admin@wmahub.ua`).
- Méthodes alternatives: `Sign in with Apple` ou `Sign in with Google` si souhaité.

Accès aux fonctionnalités principales:
- Tableau de bord: Après connexion → onglet **Accueil** (affiche statistiques, projets récents, quick actions).
- Créer un projet: **Accueil** → bouton **Créer un Projet** → remplir titre/artiste → ajouter couverture → ajouter pistes → soumettre.
- Upload de fichiers: Formats acceptés: **JPG, PNG** (couverture), **MP3, FLAC, WAV** (pistes). Taille maximale recommandée: **100 MB** par fichier.
- Distributions: Onglet **Distributions** → lister, filtrer et partager les liens.
- Assistant IA: Menu principal → **Assistant IA** (génération paroles/suggestions).
- Notifications: Icône cloche (top-right) → voir notifications récentes.
- Suppression de compte: Profil → Paramètres → **Supprimer mon compte** (confirmer avec mot de passe).

Fichiers d'exemple (pour tests manuels):
- `cover_sample.jpg` — image de couverture (format JPG, ex. 3000×3000 px)
- `sample_track.mp3` — piste audio d'exemple (MP3, 320 kbps)
- `metadata_sample.json` — exemple JSON de métadonnées (title, artist, genre, release_date)

Remarque: Nous pouvons fournir ces fichiers sur demande ou les déposer dans un dossier `uploads/sample/` si vous souhaitez les récupérer directement.

Exemples rapides (API / cURL):
1. Authentifier (email/password) → récupérer token JWT
2. POST /api/submit_project.php (multipart/form-data) avec `cover` et `tracks[]` pour téléverser un projet test
